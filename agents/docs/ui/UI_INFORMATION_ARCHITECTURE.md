# UI Information Architecture

## Hierarquia com foco em desktop

### Zona esquerda
- meta principal
- HUD compacto
- ações principais
- status modules compactos:
  - mensagem principal
  - autosave
  - combo
  - helper
  - evento ativo

### Zona central
- fazenda como foco principal
- indicadores rápidos de progresso
- feedback visual principal associado à jogabilidade

### Zona direita
- mercado
- prestígio
- progressão
- upgrades e expansão
- ajuda recolhida por padrão

## Regras da arquitetura
- o jogador deve conseguir jogar olhando principalmente para centro + esquerda
- a direita funciona como coluna de apoio e planejamento
- status transitórios devem ser curtos e escaneáveis
- textos longos devem ser recolhidos ou movidos para blocos secundários

## Decisões de UX
- ajuda começa fechada
- fazenda sobe visualmente na página
- os módulos laterais podem ser mais compactos e densos
- mobile continua empilhando, mas sem perder ordem lógica
