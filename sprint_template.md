# Sprint Template

## File Mapping

Use this template as a logical structure, distributed into the sprint files below:

- `Sprint Goal`, `In Scope`, `Out of Scope`, `Implementation Plan`, `Risks`, `QA Focus`
  - goes to `agents/planning/sprint-plans/SPRINT_<N>_PLAN.md`
- `Implementation Notes`
  - goes to `agents/planning/implementation-notes/SPRINT_<N>_IMPLEMENTATION_NOTES.md`
- `QA Report`
  - goes to `tests/reports/QA_REPORT_SPRINT_<N>.md`
- `Sprint Review`
  - goes to `agents/planning/reviews/SPRINT_<N>_REVIEW.md`

Never create these files in the repository root.

## Sprint Goal
Clear description of what the sprint is trying to achieve.

---

## In Scope

List of features or changes included.

---

## Out of Scope

Explicitly list things that should NOT be changed.

---

## Implementation Plan

Technical approach to achieve the goal.

---

## Risks

Potential technical or design risks.

---

## QA Focus

Areas that require special testing attention.

---

# Implementation Notes

Summary of actual code changes.

---

# QA Report

Results of tests performed.

- gameplay
- persistence
- UI
- regressions

---

# Sprint Review

Evaluation of the sprint outcome.

- what worked
- what did not work
- remaining risks
- next priorities
